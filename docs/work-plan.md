# 用户中心管理系统 – 工作计划

> 基于 PRD V1.4（2026-07-05）编制  
> 创建日期：2026-07-05

---

## 一、计划概览

| 项目 | 内容 |
|------|------|
| 项目名称 | 用户中心管理系统（UserCenter） |
| 总工期 | 12 周 |
| 后端技术栈 | ASP.NET Core 10 + EF Core + IdentityServer/OpenIddict |
| 前端技术栈 | React 18+ + Tailwind CSS 4（独立 SPA） |
| 数据库 | PostgreSQL / SQL Server / MySQL（EF Core 多库兼容） |
| 部署方式 | Docker 容器化部署 |

---

## 二、阶段划分总览

| 阶段 | 周期 | 主要任务 |
|------|------|----------|
| 阶段 0：项目初始化与环境搭建 | 第 1 周前半 | 仓库创建、项目脚手架、CI/CD 基础 |
| 阶段 1：技术方案设计 | 第 1 周后半 – 第 2 周 | 架构设计、数据库设计、IdentityServer/OpenIddict 选型 |
| 阶段 2：核心基础设施开发 | 第 3 – 4 周 | 数据模型、Identity 集成、JWT/OAuth2 基础、中间件 |
| 阶段 3：业务模块开发 | 第 5 – 7 周 | 用户/应用/审计/邮件/配置/管理后台 API |
| 阶段 4：用户端页面开发 | 第 6 – 8 周 | React SPA（登录/注册/授权/密码重置/个人中心） |
| 阶段 5：测试 | 第 9 – 10 周 | 功能测试、集成测试、安全测试 |
| 阶段 6：部署上线 | 第 11 周 | 生产环境部署、监控接入 |
| 阶段 7：验收交付 | 第 12 周 | 验收、文档交付 |

> 阶段 3 与阶段 4 有 2 周并行期（第 6-7 周），后端和前端可同步推进。

## 五、风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| IdentityServer/OpenIddict 选型延迟 | 阻塞认证模块开发 | 中 | 第 1 周即启动法务评审，最迟第 2 周完成决策；备选方案预研同步进行 |
| OAuth2 流程复杂导致前端联调延期 | 阶段 4 延期 | 中 | 后端优先提供 Mock 端点，前端可独立开发；第 6 周起前后端每日站会同步 |
| IMemoryCache 多实例不一致 | 缓存数据冲突 | 中 | 设置短过期时间（≤5 分钟）；必要时在第 9 周前切换到 Redis |
| 第三方邮件服务不稳定 | 邮件功能不可用 | 中 | 后台队列异步发送，失败重试 3 次；预留备用 SMTP 服务器 |
| 性能不达标（500 RPS 登录） | 需性能调优 | 低 | 第 5 周开始做初步压力测试，预留第 9 周调优时间 |
